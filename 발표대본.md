## 슬라이드 3 — 편의성: 플랫폼 기본

첫 번째, 편의성입니다.

Snowflake는 정형 데이터뿐만 아니라 JSON, Parquet, PDF, 이미지, 오디오, 비디오까지 단일 플랫폼에서 처리하는 완전 관리형 클라우드 플랫폼입니다.

핵심 기능들을 짚어보겠습니다:

- **제로 관리**: 인덱스 설계, 파티션 튜닝, 용량 산정 같은 전통적 DBA 작업이 완전히 불필요합니다. 이건 Snowflake의 기본 아키텍처 특성입니다.

- **Adaptive Compute** (GA): 이번에 정식 출시된 기능으로, 워크로드에 따라 최적 컴퓨팅을 자동으로 배분합니다. 기존 대비 약 2배 빠릅니다. 더 이상 "이 쿼리에 M을 쓸까, L을 쓸까?" 고민할 필요가 없어진 겁니다.

- **멀티 클라우드**: AWS, Azure, GCP 어디서든 동일한 경험을 제공하는 기본 기능입니다.

- **Code Bundles** (Public Preview): Python이나 Java 파일을 Stored Procedure로 래핑하지 않고 Snowflake에서 바로 실행할 수 있습니다. 개발 편의성이 크게 향상됩니다.

- **Snowflake AIM** (GA): AI 기반 마이그레이션 도구입니다. SnowConvert AI, Snowpark Migration Accelerator, Datometry를 하나로 통합했고, 코드 마이그레이션을 평균 88% 가속합니다.

- **Snowflake Postgres** (GA): 완전 관리형 PostgreSQL 서비스입니다. 올해 2월에 GA되었고, 이번에 Private Link과 고객 관리 키가 추가되었습니다.

- **Postgres Data Mirroring** (Public Preview): Postgres 테이블을 Snowflake로 실시간 CDC 복제하는 기능입니다. 별도 CDC 파이프라인 구축이 불필요합니다.

핵심 메시지는, 인프라 관리에 쓰던 시간을 데이터 활용과 AI 전략에 집중하라는 것입니다.

## 슬라이드 4 — 편의성: CoCo

이번 Summit의 최대 화제, **Snowflake CoCo**입니다.

기존에 Cortex Code라고 불렸는데, 이번에 공식적으로 CoCo로 리브랜딩되었습니다. 개발자를 위한 AI 코딩 에이전트입니다.

- **자연어로 전체 개발 워크플로우를 자율 수행**합니다. 데이터 파이프라인 구축, SQL 작성, dbt 빌드, 앱 배포까지 말로 시키면 됩니다.

- **Cloud Agents** (GA): 로컬에 아무것도 설치하지 않아도 Snowsight 브라우저에서 전체 개발 워크플로우를 실행할 수 있습니다. 셸 명령, Python, 패키지 설치, dbt 빌드, 웹 브라우징까지 클라우드 샌드박스에서 돌아갑니다.

- **CoCo Desktop** (GA): 데스크톱 앱으로 Plan → Build → Deploy를 하나의 흐름으로 처리합니다. VS Code Extension, Excel Extension, Slackbot까지 어디서든 접근 가능합니다.

- **Snap and Ask** (Private Preview): 화면에서 차트나 오류 메시지를 캡처하고 "이게 왜 이래?"라고 물어보면, CoCo가 시각적 컨텍스트를 이해하고 데이터 소스를 추적해서 원인을 알려줍니다.

- **Skill Catalog** (Public Preview): 팀에서 만든 스킬과 플러그인을 조직 전체에서 검색하고 재사용할 수 있는 카탈로그입니다.

중요한 점은, **파괴적 명령에는 반드시 사용자 승인이 필요**합니다. AI가 단독으로 데이터를 삭제하거나 변경하지 않습니다. 이게 엔터프라이즈 환경에서 안심하고 쓸 수 있는 이유입니다.

Christian Kleinerman의 말처럼, "우리는 마찰 제거 사업을 하고 있습니다."

## 슬라이드 5 — 편의성: CoWork

**Snowflake CoWork** — 기존 Snowflake Intelligence에서 리브랜딩되었습니다.

CoCo가 개발자를 위한 것이라면, CoWork는 **모든 직원을 위한 개인 AI 업무 엔진**입니다.

- **Personal Work Engine** (Private Preview): 기존에는 "어떤 에이전트를 써야 하지?"를 고민해야 했는데, 이제 하나의 개인 에이전트가 알아서 적합한 에이전트를 조율합니다. CEO든 현장 직원이든 같은 인터페이스에서 각자 수준에 맞는 답변을 받습니다.

- **User Memory** (Private Preview): CoWork가 사용 패턴과 선호도를 학습합니다. 쓸수록 똑똑해지는 개인화 엔진입니다.

- **Deep Research** (GA): 복잡한 질문에 대해 병렬 서브 에이전트가 동시에 조사하고, 심층 분석 보고서를 자동 생성합니다.

- **Artifacts** (GA): 기존의 정적 PDF 보고서가 아니라, 실시간 데이터와 연결된 살아있는 대시보드를 자동 생성하고 조직 전체에 공유합니다. 거버넌스 정책이 자동 적용되어 권한 없는 사람은 볼 수 없습니다.

- **Scheduled Tasks** (Private Preview): "매주 월요일 6시에 이 분석을 이메일로 보내줘"라고 말하면 끝입니다. 자연어로 반복 업무를 예약합니다. 아직 Private Preview이지만 방향성이 매우 기대됩니다.

삼성전자가 이미 임직원 1,000명이 매일 CoWork를 활용하고 있고, 이 중 데이터 과학자는 단 한 명도 없습니다. 이것이 CoWork의 가치입니다.

## 슬라이드 6 — 연결성: 데이터 공유

두 번째, 연결성입니다.

Snowflake의 핵심 강점 중 하나가 데이터 공유와 개방형 아키텍처입니다.

- **Secure Data Sharing** (GA): 데이터를 복사하지 않고 계정 간 즉시 공유합니다. Zero-Copy라서 지연 없고 추가 비용도 없습니다.

- **Open Sharing** (Public Preview): 이번에 새로 나온 기능으로, Snowflake를 사용하지 않는 외부 조직에도 Iceberg REST Catalog API를 통해 데이터를 공유할 수 있습니다. "Snowflake는 닫힌 생태계"라는 비판에 대한 완벽한 응답입니다.

- **Data Resharing** (GA): 공유받은 데이터를 제3자에게 재공유할 수 있습니다.

- **Snowflake Marketplace** (GA): 3,300개 이상의 라이브 데이터셋과 서비스를 즉시 구독 가능합니다.

- **Apache Iceberg v3** (GA): Snowflake가 업계에서 가장 광범위한 V3 스펙을 구현했고, V4 개발에도 참여하고 있습니다. 삭제 벡터, 행 리니지, Variant 타입 등을 지원합니다.

- **Snowflake App Runtime** (Public Preview): Node.js나 React 앱을 Snowflake 보안 경계 내에서 바로 호스팅합니다. `snow app deploy` 한 줄이면 배포 끝입니다.

- **Streamlit in Snowflake** (GA): 컨테이너 런타임 버전이 GA되었고, Git 통합과 워크스페이스를 지원합니다.

## 슬라이드 7 — 연결성: Zero-Copy & 스트리밍

Zero-Copy 통합과 스트리밍 관련 발표입니다.

- **SAP Zero-copy** (GA): 가장 많이 요청된 파트너 통합이 드디어 정식 출시되었습니다. ETL 없이 SAP 데이터를 Snowflake에서 직접 쿼리합니다.

- **Workday Data Cloud** (Public Preview): HR과 재무 데이터를 이동 없이 직접 쿼리합니다.

- **IBM watsonx.data** (Public Preview): DB2와 메인프레임 데이터를 직접 연동합니다.

- **AVEVA Connect** (Public Preview): 제조 현장의 OT 데이터, 공장 설비 데이터를 직접 통합합니다.

- **Snowflake Datastream** (Private Preview): 이번 Summit의 인프라 분야 최대 발표입니다. 완전 관리형, Kafka Wire Protocol 호환 스트리밍 서비스입니다. 기존 Kafka 클라이언트의 설정만 바꾸면 Sub-second 지연으로 데이터가 Snowflake에 들어옵니다. 브로커, 파티션, 오프셋 관리가 전부 불필요합니다. Snowflake 거버넌스와 리니지가 자동 적용됩니다.

- **Snowflake Openflow** (GA): Apache NiFi 기반의 완전 관리형 데이터 통합 서비스입니다. Oracle 커넥터는 GA, Veeva/BigQuery/MongoDB는 Public Preview입니다.

- **Query Across External Sources** (Private Preview): CoWork에서 Redshift나 Postgres 같은 외부 소스에 직접 AI 쿼리를 실행할 수 있습니다.

## 슬라이드 8 — 연결성: 협업 & MCP

- **Multi-party Collaboration** (GA): 여러 조직이 하나의 보안 환경에서 각자의 데이터를 제공하고 분석하는 협업 체계입니다. Netflix가 얼리 어답터로 사용 중입니다.

- **Natoma 인수**: 이번 Summit에서 확인된 인수 건입니다. Natoma는 엔터프라이즈 MCP(Model Context Protocol) 플랫폼으로, 약 100개의 비즈니스 시스템 커넥터를 제공합니다. Slack, Gmail, Jira, GitHub, Zoom, Google Drive, Microsoft 365 등과 연결됩니다. AI 에이전트가 외부 시스템에서 작업할 때도 Snowflake의 거버넌스가 적용됩니다.

- **Open Semantic Interchange (OSI)**: 54개 벤더가 참여하는 시맨틱 모델 공유 개방 표준입니다.

현재 Snowflake 파트너는 700개 이상이며, Fivetran, dbt, Tableau, Power BI, Anthropic 등이 포함됩니다.

## 슬라이드 9 — 신뢰성: 보안 & 컴플라이언스

세 번째, 신뢰성입니다.

기본 보안 아키텍처부터 짚겠습니다:

- **상시 암호화** (기본): End-to-End AES-256 암호화와 Tri-Secret Secure(고객이 키를 관리)를 제공합니다.

- **컴플라이언스 인증** (기본): SOC 2 Type II, HIPAA, FedRAMP, ISO 27001, PCI DSS를 보유하고 있습니다.

- **Dynamic Data Masking과 Row Access Policy** (GA): 컬럼 단위, 행 단위로 세분화된 접근 제어를 합니다.

이번 Summit에서 특히 주목할 보안 기능들:

- **AI Agent Identity** (GA): 모든 AI 에이전트에 암호화 기반 고유 신원을 부여합니다. 에이전트별로 RBAC을 적용하고 전체 활동을 감사 추적합니다. 전통적 보안은 인간 사용자를 위해 설계되었는데, 에이전트 시대에는 이것이 필수입니다.

- **Horizon AI Guardrails** (GA): CoCo와 CoWork에서 프롬프트 인젝션과 탈옥 시도를, 제로데이 공격까지 포함해서 실시간으로 차단합니다.

- **Multi-Party Approval** (Public Preview): MFA를 조직 전체에서 비활성화하는 것처럼 최고 위험 작업에는 2인 이상 승인이 필요합니다. 내부자 위협이나 에이전트 오작동을 방지합니다.

- **Data Movement Policies** (Private Preview): 특정 태그가 부여된 데이터의 외부 반출을 시스템 레벨에서 차단합니다. Stage로 내보내기도, Snowsight에서 다운로드도 불가합니다.

- **AI Security Posture Management** (Public Preview): AI 에이전트의 보안 설정을 지속적으로 모니터링하고, 프롬프트 인젝션과 탈옥을 방어합니다.

## 슬라이드 10 — 신뢰성: 거버넌스 & 옵저버빌리티

거버넌스 영역입니다.

- **Horizon Catalog** (GA): 데이터 자산의 분류, 태깅, 리니지 추적을 통합 관리합니다.

- **Horizon Context** (Private Preview): PostgreSQL, SQL Server, Tableau, Power BI, dbt 등 외부 소스에서 메타데이터를 자동 수집하고, AI 문서를 자동 생성하고, 시맨틱 뷰를 자동 탐색합니다. 이게 있어야 AI 에이전트가 비즈니스 맥락을 이해합니다.

- **Cortex Sense** (Private Preview): 이번 키노트의 핵심 수치입니다. 쿼리 이력, 대시보드, 메타데이터로부터 런타임에 맥락을 자동으로 조합해서, 맥락 없는 모델이 24%의 정확도를 보일 때 **86%의 정확도**를 달성합니다. 모델이 아닌 컨텍스트가 정확도를 결정한다는 증거입니다.

- **Intent-Driven Governance** (Private Preview): 자연어로 "내 DB의 모든 PII를 보호해줘"라고 말하면, 자동으로 분류하고 정책을 적용합니다. SQL로 하나하나 마스킹 정책을 짤 필요가 없어집니다.

- **Trust Center** (GA): 보안 상태를 실시간으로 스캔하고 권고합니다. 데이터 유출 탐지 기능이 Public Preview로 추가되었습니다.

- **Observe by Snowflake** (GA): 인수한 Observe를 통합해서 로그, APM, 인프라 모니터링을 Snowflake 네이티브로 제공합니다. Iceberg 테이블 기반이고, CoCo에서 직접 조작 가능합니다.

Thomson Reuters의 Caitlin Hugerty VP는 이렇게 말했습니다: "거버넌스는 제약이 아닌 가속 장치입니다."

## 슬라이드 11 — 신뢰성: 비즈니스 연속성

재해 복구와 비즈니스 연속성입니다.

- **Time Travel** (GA): 최대 90일까지 데이터를 과거 시점으로 복구할 수 있습니다.

- **Fail-safe** (GA): Time Travel 이후 7일간 추가 보호 계층을 제공합니다.

- **차세대 계정 복제** (GA): 로그 기반으로 기존 대비 약 20배 빠른 복제를 지원하며, SLA 기반 RPO를 보증합니다.

- **교차 리전/교차 클라우드 복제** (기본): 재해 발생 시 자동 페일오버됩니다. 실제로 2025년 클라우드 장애 때 300개 이상 고객의 워크로드가 무중단 전환되었습니다.

- **99.9% SLA** (기본): Business Critical Edition 이상에서 보장합니다.

규제 산업을 포함한 모든 환경에서 안정적 운영을 보장합니다.

## 슬라이드 12 — 경제성

네 번째, 경제성입니다.

Snowflake는 사용한 만큼만 과금되는 소비 기반 모델입니다.

- **컴퓨팅/스토리지 분리 과금** (기본): 쿼리하지 않을 때 컴퓨팅 비용은 0원입니다.

- **Adaptive Compute** (GA): 과잉 프로비저닝을 원천적으로 제거합니다.

- **자동 일시 중지/재개** (기본): 유휴 시 자동으로 중지되어 불필요한 비용을 방지합니다.

- **압축 스토리지** (기본): 평균 4~7배 압축으로 저장 비용을 절감합니다.

이번에 새로 나온 비용 거버넌스 기능들:

- **AI Cost Controls** (GA): AI 사용량을 기존 예산 체계와 연동합니다. 사용자별 할당량 설정, 부서별 비용 귀속이 가능하고, 예산 초과 시 자동으로 프로시저가 실행됩니다.

- **Budget Custom Actions** (GA): 지출 임계값에 도달하면 저장 프로시저가 자동으로 트리거됩니다. 예를 들어 "80%에 도달하면 알림, 100%에 도달하면 웨어하우스 중지"를 자동화할 수 있습니다.

- **Serverless 확대** (GA): Snowpipe Streaming, Dynamic Tables, Datastream 등이 서버리스 기반이라 관리 비용 자체가 없습니다.

핵심은, 초기 대규모 투자 없이 시작하고 사용량에 따라 선형적으로 확장된다는 점입니다.

## 슬라이드 13 — AI 혁신: Cortex AI 플랫폼

다섯 번째, AI 혁신입니다.

Snowflake의 핵심 비전은 "Agentic Enterprise" — 에이전트형 기업입니다. AI 모델 자체가 아닌, 기업 고유 데이터와 맥락의 결합이 진짜 경쟁력이라는 것입니다.

- **AI_COMPLETE 멀티모달 확장** (GA): SQL 함수 하나로 텍스트, 이미지, PDF는 물론 이제 오디오와 비디오까지 처리합니다.

- **Cortex AI Function Studio** (Public Preview): 기업 전용 AI 함수를 만들고, 평가하고, 배포하는 플랫폼입니다. 팀마다 제각각인 프롬프트를 표준화하고 거버넌스를 확보할 수 있습니다.

- **Cortex Training** (Private Preview): Snowflake 안에서 데이터 이동 없이 Qwen, Mistral 같은 오픈 웨이트 모델을 파인튜닝하고 강화학습합니다. 동일 GPU 대비 약 2배의 학습 효율을 제공합니다.

- **Agentic Search** (Public Preview): 일반적인 검색은 관련 문서를 보여주지만, Agentic Search는 비정형 문서에서 구조화된 팩트를 추출한 다음 분석 쿼리를 실행해서 정확한 수치로 응답합니다.

- **Grok (xAI)** (신규): Anthropic Claude, OpenAI GPT, Google Gemini, Meta Llama, Mistral, DeepSeek에 이어 xAI의 Grok이 탑재되었습니다. 현재 7개 이상의 모델 벤더를 지원합니다.

## 슬라이드 14 — AI 혁신: 에이전트 인프라 & ML

에이전트 인프라와 머신러닝, 그리고 성능 관련 발표입니다.

**에이전트 인프라:**

- **Skill Catalog** (Public Preview): 검증된 에이전트 스킬을 조직 전체에서 검색하고 재사용합니다.

- **Cortex Agents + MCP** (GA): 외부 도구(Slack, Jira, CRM 등)와 연결하는 AI 에이전트를 Snowflake 거버넌스 하에 구축합니다.

- **Semantic View Autopilot** (GA): 기존 SQL, Tableau, Power BI 자산으로부터 시맨틱 뷰를 자동 생성합니다. 수동으로 하나하나 만들 필요가 없어졌습니다.

- **Pipeline Builder** (Private Preview): ML 학습 파이프라인을 시각적 DAG로 편집하고 모니터링합니다.

**머신러닝:**

- **Online A/B Testing** (Public Preview): 실시간 트래픽에서 모델 실험을 수행하고 승자를 자동 승격합니다.

- **Streaming Feature Serving** (GA): 약 10ms 지연으로 온라인 피처 벡터를 서빙하며, 2초 미만의 데이터 신선도를 보장합니다.

**성능:**

- **Interactive Compiler** (GA): 새로운 쿼리 컴파일러로 컴파일 속도가 40배 향상됐습니다. 전체 워크로드가 3~4배 가속됩니다.

- **Unistore Engine Optimization** (Public Preview): Hybrid Table의 지연과 처리량이 8배 개선됐습니다.

핵심 메시지는, 별도 ML 인프라 없이 Snowflake 안에서 데이터 수집부터 학습, 배포, 서빙, 모니터링까지 AI 전체 라이프사이클을 수행할 수 있다는 것입니다.
